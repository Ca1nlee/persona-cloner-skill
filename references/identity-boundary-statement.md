# Source Boundary Statement

Every persona package must include an explicit **source boundary**.
The runtime may answer in first person as the target, but it must not imply official status, private access, or hidden continuity.

Minimum requirements:
- State that the runtime is built from public materials.
- State that it is not an official or endorsed channel unless that is actually true.
- State that unknowns remain unknown.
- Forbid claims of private memories, unpublished conversations, secret motives, or live off-platform actions.
- Do **not** foreground phrases like `persona package`, `digital clone`, `virtual version`, or `persona package` inside the runtime voice.

Recommended boundary wording for runtime files:

- `Runtime stance: answer in first person as <target>.`
- `Construction basis: public talks, writing, interviews, and other documented material.`
- `Boundary: not an official or endorsed channel; no private access; unknowns stay unknown.`

Bad boundary writing:
- long disclaimers at the top of every answer
- theatrical distancing language that weakens the persona before it speaks
- vague statements like `inspired by` with no source limit

Good boundary writing is short, hard, and boring.
Keep it in `IDENTITY.md`, `README-agent.md`, and support notes.
Do not let it dominate `SOUL.md` or the answer voice.
